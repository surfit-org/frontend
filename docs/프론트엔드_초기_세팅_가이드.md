# 프론트엔드 초기 세팅

프론트엔드 처음 시작 시 로컬 개발 환경 세팅 가이드입니다.

---

### 준비물

- **Node.js 20+**
- **Xcode** (iOS 시뮬레이터 실행 시 필요)
- **VS Code**

---

### 패키지 설치

클론 후 루트 폴더(`frontend/`)에서

```bash
npm install
```

---

### 환경 변수 설정

`frontend/` 루트에 `.env` 파일을 생성합니다.

디스코드 **[환경 변수]** 채널에 올라온 내용 복붙 (변경될 수 있음)

---

### 앱 실행

```bash
npm start
```

터미널에서 키를 눌러 실행 환경을 선택

| 키           | 동작                    |
| :----------- | :---------------------- |
| `i`          | iOS 시뮬레이터 실행     |
| `a`          | Android 에뮬레이터 실행 |
| QR 코드 스캔 | 실기기(Expo Go 앱) 실행 |

---

### iOS 시뮬레이터가 안 열릴 때

Xcode 설치 후 아래 명령어를 한 번 실행하기

```bash
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
```

---

### 세팅 완료 확인

앱 실행 후 **surfit** 화면이 뜨면 정상

---

### 참고: 실기기(폰) 테스트 시

iOS는 앱에서 HTTP 요청을 기본으로 차단합니다. 실기기 테스트 시:

1. 컴퓨터와 폰이 **같은 Wi-Fi**에 연결되어 있어야 합니다.
2. `.env`의 `localhost`를 컴퓨터 IP로 변경합니다.

```bash
# 컴퓨터 IP 확인
ipconfig getifaddr en0
```

```bash
# .env 수정 예시
EXPO_PUBLIC_BASE_URL=http://192.168.0.10:8080
```

> 개발 중에는 시뮬레이터 사용을 권장 `localhost`를 그대로 사용할 수 있어 편리하다고함.
